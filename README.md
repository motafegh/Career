## State and document rules

- Plans and specifications define requirements and pass conditions.
- The tracker and evidence records state whether requirements were satisfied.
- Session evidence records what happened; it does not replace the tracker.
- README and `AGENTS.md` files must not carry exact session-level next actions or duplicate current method/gate details.
- Stable governance files must not require routine edits when one test, session, sub-gate, or next action changes.
- A consequential decision may require an ADR and a tracker result; it does not automatically require repository-wide status propagation.
