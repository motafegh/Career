# 90-Day Career Plan

**Status:** Controlling career-level plan  
**Owner:** Ali Rajabi  
**Execution period:** 2026-07-20 to 2026-10-17  
**Responsibility:** Career direction, flagship allocation, formal review points, and Day-90 career evidence

## 1. Working identity

> **AI-augmented Python/data/ML engineer developing secure engineering capability.**

This identity remains fixed until the Day-90 review unless Ali explicitly authorizes a formal career-strategy review.

## 2. Primary allocation

UpgradePilot is the selected primary flagship project for the execution period.

Career records only the durable allocation and coarse reviewed state. UpgradePilot owns:

- product mission and boundary;
- project operating guide;
- detailed route and milestones;
- current plans and continuation;
- technical specifications and ADRs;
- source, tests, evidence, and project-local decisions.

Career is not updated for ordinary UpgradePilot tests, commits, sessions, sub-gates, or next actions.

## 3. Career capability priorities

1. Python application structure and implementation ownership.
2. Testing and evidence-based debugging.
3. Real-input acquisition, parsing, validation, and provenance.
4. Data modeling, persistence, and SQL.
5. Reproducible end-to-end pipelines and user-facing operation.
6. Secure engineering and trustworthy evidence handling.
7. Baselines, evaluation, and honest ML/AI experimentation.
8. Project-connected advanced-system exposure.
9. Technical explanation, portfolio evidence, and job-market calibration.

## 4. Role direction

Primary role families:

- Junior Python Developer;
- Junior Data Engineer;
- Python Data/Automation Engineer;
- Applied AI or AI Application Engineer;
- AI Implementation Engineer;
- Junior Backend Engineer with data-heavy work;
- Technical Implementation or Solutions Engineer using Python;
- Data Analyst with meaningful Python and SQL engineering;
- ML engineering trainee, internship, or junior role.

Secondary exploration may include security automation, test automation, MLOps, cloud/platform, developer tooling, and distributed systems when evidence supports it.

Exposure does not create a professional title or mastery claim.

## 5. Advanced-system commitment

The approved minimum remains:

- A1 hands-on exposure for distributed queues, advanced MLOps, microservices, Kubernetes, multi-cloud, and bounded autonomous multi-agent systems;
- at least two A2 project-integrated pilots unless an explicit strategic review changes the requirement;
- A3 permanent adoption only when evidence justifies it.

Detailed implementation and evidence belong to UpgradePilot. Career evaluates the resulting evidence only during an explicit review.

## 6. Formal review points

- Day 14 workload calibration: 2026-08-02;
- Day 30 review: 2026-08-18;
- Day 60 review: 2026-09-17;
- Day 90 review: 2026-10-17;
- any additional review explicitly requested by Ali.

A formal review may inspect UpgradePilot source, tests, plans, memory, working evidence, and portfolio artifacts as needed.

## 7. Career review outputs

A Career review may update:

- coarse UpgradePilot state;
- assessed capability depth and transfer limits;
- workload/capacity decision;
- role targeting;
- portfolio claim boundaries;
- advanced-system strategy or requirements when explicitly in scope;
- whether the 90-day career route should continue, narrow, or change.

It must not copy ordinary project continuation into Career.

## 8. Day-90 evidence

Career closure should be based on:

- a bounded runnable UpgradePilot core;
- honest capability and assistance assessment;
- reproducible source/test/evaluation evidence;
- explicit limitations and rejected methods;
- project-connected advanced-system evidence under the approved strategy;
- a portfolio package with defensible claims;
- calibration against representative job descriptions.

## 9. Change control

Change this plan only when career identity, primary project allocation, capability priorities, role direction, formal review schedule, advanced-system commitments, or Day-90 career evidence requirements change.

Do not update it for ordinary project progress.
