# Program Monthly Plan

The 90 days use three exact 30-day program months rather than calendar months.

Each month contains **26 Monday–Saturday production days**, producing a standard target of **104 focused hours** before recorded Red-mode exceptions.

## Month 1 — Close foundations and complete the first Python vertical slice

**Dates:** 2026-07-20 to 2026-08-18  
**Days:** 1–30  
**Standard focused time:** 104 hours

### Required outcomes

1. Close the namespace SSH checkpoint during Week 1.
2. Preserve normal and failed-authentication fixtures and truth records.
3. Establish an importable Python package and CLI.
4. Implement normalized authentication-event models.
5. Implement parsing, validation, and quarantine.
6. Establish SQLite persistence and foundational SQL.
7. Implement the time-window detector and finding/report path.
8. Pass one clean raw-evidence-to-report integration command before the Day 30 review.
9. Demonstrate parser modification, threshold modification, SQL querying, and one changed-case diagnosis.

### Day 14 workload review — 2026-08-02

Judge only delivery density and workload realism:

- Did Week 1 close namespace work and produce the Python intake foundation?
- Did Ali average near the 24-hour weekly standard when functioning allowed?
- Were trivial tasks completed quickly rather than padded?
- Are deliverables technically challenging enough?
- Is workload too low, realistic, or unsafely high?

The Day 14 review may compress or rebalance exact deliverables. It may not change identity or project.

### Day 30 review — 2026-08-18

Judge:

- Does a clean command perform raw → parse → validate/quarantine → persist → detect → report?
- Can Ali explain and modify the parser and detector threshold?
- Can Ali write and explain required SQLite queries?
- Are invalid, malformed, duplicate, and below-threshold cases tested?
- Did implementation dominate documentation?
- Were 24-hour weeks normally achieved or honestly capacity-adjusted?

### Month 1 failure response

If the complete vertical slice is incomplete:

- cut presentation work;
- carry only the smallest blocking technical gate;
- do not begin Docker until raw-to-report behavior passes;
- do not reopen namespace scope or create another project.
